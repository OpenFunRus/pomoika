# Былины и Мемы — Геймдизайн-документация (GDD) v1.0

> **Жанр:** Roguelite / Bullet Heaven / Vampire Survivors-like  
> **Вид:** Top-down, pixel-art 32×32  
> **Движок:** Godot 4.6 (GL Compatibility)  
> **Референсы:** Vampire Survivors, 20 Minutes Till Dawn, Halls of Torment  
> **Ключевая фишка:** Юмористическая русская деревенская мифология, разблокировка персонажей через победу над ними как супербоссами.

---

## 1. Видение игры (Game Vision)

### 1.1. Элеватор-питч
«Дед отбивается от одержимых куриц на огороде. Чем дольше выживает — тем больше открывает родственников и домашних животных с уникальным оружием. Каждый забег — это эпизод деревенского эпоса с комикс-вставками и мемным юмором.»

### 1.2. Эмоциональные столпы
1. **Ностальгия по деревне** — заборы, грядки, ёлки, сорняки, колодец.
2. **Абсурдный юмор** — бабка с метлой, внучка с водяным пистолетом, кошка с комками шерсти.
3. **Прогрессия «семейного древа»** — каждый новый герой открывает новую ветку механик и врагов.
4. **Лёгкий стресс → катарсис** — 10-минутные забеги, понятные правила, визуальный хаос с контролем.

### 1.3. Целевая аудитория
- **Core:** фанаты Vampire Survivors и мобильных survivor-игр.
- **Casual:** игроки 25–40 лет, ценящие пиксель-арт и постсоветский колорит.
- **Платформы:** PC (Steam), потенциально Android/iOS (с адаптированным UI).

## 3. Критические пробелы (What’s Missing)

### 3.1. Геймплейные (блокеры веселья)
| Пробел | Почему это важно | Критичность |
|--------|------------------|-------------|
| **Нет экрана Level-Up** | В VS-like жанре это ядро цикла обратной связи. Сейчас уровни просто присваиваются. | 🔴 Критично |
| **Нет смерти / Game Over** | Нет риска = нет напряжения. HP уходит в минус, забег бесконечен. | 🔴 Критично |
| **Нет боссов и супербоссов** | Нет точек climax в забеге. Нет механики разблокировки новых героев. | 🔴 Критично |
| **Нет кат-сцен / комиксов** | Это ключевая фишка концепта (3 плашки). Без неё теряется «Былины» в названии. | 🟡 Высоко |
| **Одно оружие у деда** | VS-like предполагает комбо из 6+ слотов оружия/пассивок. | 🟡 Высоко |
| **Нет возврата в меню из игры** | Базовый UX. Сейчас только Alt+F4. | 🟡 Высоко |
| **Нет паузы** | Базовый UX. | 🟢 Средне |
| **Нет звука и музыки** | Атмосфера деревни требует звуков кур, петухов, баяна. | 🟢 Средне |

### 3.2. Технические (долг)
| Пробел | Риск | Критичность |
|--------|------|-------------|
| **Нет Autoload-синглтонов** | `class_name` + ручное создание — сложно масштабировать. | 🟡 Высоко |
| **Game.gd ~760 строк** | God-класс, смешение генерации мира, HUD, спавна, XP. | 🟡 Высоко |
| **HeroCatalog не связан с Player** | Выбор героя сохраняется, но в игре всегда Дед. | 🔴 Критично |
| **Нет системы частиц подбора лута** | Визуальная обратная связь слабая. | 🟢 Средне |
| **Сейв не хранит состояние забега** | Нельзя сделать «Продолжить» из главного меню по-настоящему. | 🟢 Средне |

### 3.3. Контентные
| Пробел | Описание |
|--------|----------|
| **Нет второй локации** | После открытия 2–3 героев нужна новая карта (лес, река, поле). |
| **Нет мета-прокачки** | Кнопки есть, но disabled. Нужны: урон, скорость, HP, магнит, удача, рост. |
| **Нет ачивок** | Заглушка в меню. Ачивки = направление для новичка. |
| **Нет системы «удачи»** | Влияет на редкость дропа и апгрейдов. Важно для реиграбельности. |

## 4. Лучшие подходы к реализации (Best Practices)

### 4.1. Архитектура кода
**Проблема:** God-класс `game.gd` + отсутствие Autoload.

**Решение — внедрить 4 синглтона (Autoload):**
```
GameManager      — глобальное состояние (сцена, пауза, сейв).
SaveManager      — обёртка над JSON, шифрование/валидация.
AudioManager     — банки звуков, музыка, настройки громкости.
EventBus         — сигналы глобальных событий (level_up, enemy_died, coin_gained).
```

**Разбить Game.gd на модули:**
```
WorldGenerator   — TileMap, забор, грядки, объекты.
EnemyDirector    — спавн кур, лимиты, point-mode, волны.
HiveDirector     — ульи и муравейники (отдельно от глобального спавна).
RunProgress      — XP, уровни, таймер, условия босса.
RunHUD           — CanvasLayer, XP-bar, индикаторы, уроные цифры.
```

### 4.2. Система оружия (Weapon System)
**Проблема:** Сейчас только топор. Нужно 6 слотов.

**Решение — компонентный подход:**
- `WeaponSlot` — узел-контейнер в Player.
- `BaseWeapon` — абстрактный класс с методами `_fire()`, `_upgrade(level)`.
- Каждое оружие = отдельная сцена: `AxeThrower`, `ScytheSpinner`, `BroomCone`.
- **Пассивные предметы** — отдельные слоты (рост, удача, скорость, магнит, броня, урон).

**Пример матрицы оружия:**
| Герой | Оружие 1 | Оружие 2 | Оружие 3 | Пассивная тема |
|-------|----------|----------|----------|----------------|
| Дед | Топор (метательный) | Коса (вокруг) | ??? | Сила |
| Бабка | Метла (конус) | Тарелка (AoE при попадании) | Пирожок (снаряд) | Скорость |
| Внучка | Водяной пистолет | Кнут (конус) | Ракетка (орбита) | Дальность |
| Жучка | Блохи (спавн) | Вонь (облако) | Цепь (кнут) | Статус-эффекты |
| Кошка | Шерсть (снаряд) | Когти (ближний) | Хвост (AoE вокруг) | Ловкость |
| Мышка | Мышеловки (ловушки) | Орех (отскок) | ??? | Удача |

### 4.3. Экран Level-Up (Upgrade Screen)
**Референс:** Vampire Survivors — 3 карточки на выбор.

**Рекомендуемая архитектура:**
- При достижении порога XP — `get_tree().paused = true` (или кастомная пауза, не останавливающая UI).
- `UpgradePool` — синглтон/ресурс со всеми возможными апгрейдами.
- **Веса выпадения:**
  - 60% — оружие/пассивка, уже имеющиеся (для эволюции).
  - 30% — новые предметы из пула героя.
  - 10% — редкие универсальные (крит, восстановление HP).
- **Эволюция оружия:** при максимуме оружия (уровень 8) + максимум связанной пассивки → эволюция (например, Топор → Зажигательный топор при пассивке «Спички»).

### 4.4. Босс / Супербосс цикл
**Проблема:** Сейчас нет условий появления босса.

**Рекомендуемый цикл забега:**
```
0:00 — Старт, волна 1 (курицы)
2:00 — Волна 2 (+ муравьи/пчёлы активнее)
4:00 — Мини-босс (Петух / Утка / Баран)
6:00 — Волна 3 (новые цвета, больше мобов)
8:00 — БОСС (Петух для этапа 1)
→ Убийство босса → если герой «Бабка» не открыт → СУПЕРБОСС «Бабка»
→ Убийство супербосса → разблокировка Бабки + комикс
→ Продолжение забега за Дедом (или выход в меню)
```

**Супербосс = будущий герой:**
- Использует свои будущие способности (Бабка летит на метле, кидает тарелки).
- У неё HP-bar с именем.
- После победы — короткий комикс (3 панели) + ачивка.

### 4.5. Мета-прогрессия (Meta-Progression)
**Валюта:** Золотые яйца / монеты / «Деревенские очки» (уже есть `coins` и `total_points`).

**Дерево прокачки (перманентное):**
| Узел | Макс уровень | Эффект | Стоимость |
|------|--------------|--------|-----------|
| Сила | 10 | +5% урона | 100×уровень |
| Здоровье | 10 | +10 HP | 100×уровень |
| Скорость | 5 | +5% скорости | 150×уровень |
| Удача | 5 | +3% редкого дропа | 200×уровень |
| Магнит | 5 | +10% радиуса магнита | 150×уровень |
| Рост | 5 | +5% размера снарядов | 150×уровень |
| Богатство | 5 | +10% монет с забега | 200×уровень |

**Разблокировка героев:**
- Бабка — победить супербосса «Бабку».
- Внучка — победить супербосса «Внучку» (открывается при игре за Бабку).
- Жучка — собрать 100 жал (глобальный счётчик).
- Кошка — собрать 100 кусков шерсти.
- Мышка — секретное условие (убить 1000 мышей или найти сыр на карте).

## 5. Детальный геймдизайн

### 5.1. Игровой цикл (Run Loop)
```
[Главное меню]
    ↓
[Выбор героя] ← открытые персонажи + их оружие
    ↓
[Комикс-вставка] (только при первой игре за героя)
    ↓
[Забег 10–15 минут]
    ├── Волны врагов (0:00–8:00)
    ├── Лут → XP → Level-Up (выбор апгрейдов)
    ├── Босс (8:00 или N убийств)
    ├── Супербосс (при первом прохождении этапа)
    └── Комикс + разблокировка
    ↓
[Результаты] — убийства, время, монеты, ачивки
    ↓
[Мета-меню] — прокачка, новый герой, новый забег
```

## 7. UI/UX Рекомендации

### 7.1. Главное меню
- **Фон:** живой огород (процедурная генерация без мобов, медленно движущиеся облака).
- **Кнопки:** крупные, пиксель-арт стиль, Press Start 2P.
- **Социалки:** Discord, Telegram — иконки в углу.
- **«Нашли баг»:** кнопка, открывающая форму (или ссылка на Google Forms / GitHub Issues).

### 7.2. HUD в забеге
- **XP-bar:** верхний, яркий, с анимацией заполнения.
- **HP-bar:** над персонажем (как сейчас) + крупный в углу при <30%.
- **Таймер:** обратный отсчёт до босса или просто время.
- **Убийства:** счётчик (для ачивок).
- **Уроные цифры:** всплывающие числа при уроне врагам (крит = жёлтый/красный, крупный).
- **Ачивки:** всплывающие плашки справа-снизу (как в ТЗ).

### 7.3. Экран Level-Up
- **Фон:** затемнение игры (не полная пауза — мобы продолжают идти, но игрок неуязвим).
- **3 карточки:** иконка, название, описание, редкость (цвет рамки).
- **Кнопки снизу:** «Реролл» (1 раз за забег, потом платно), «Пропустить» (если ничего не нужно).

### 7.4. Комиксы
- **Формат:** 3 горизонтальные панели (или вертикальные для мобильных).
- **Анимация:** плавное появление (alpha 0→1), можно тап/клик для ускорения.
- **Содержание:** диалоги в стиле «бабка vs дед», мемные фразы, сетап для следующего этапа.

### 8.5. Критический совет: Object Pooling
Сейчас: `instantiate()` + `queue_free()` при смерти.  
**Проблема:** при 128 мобах + лут + частицы — GC-просадки.

**Решение:**
```gdscript
class_name EnemyPool extends Node
var _pools: Dictionary = {}
func get_enemy(type: String) -> CharacterBody2D:
    var pool = _pools.get(type, [])
    if pool.is_empty(): return load("res://...").instantiate()
    return pool.pop_back()
func return_enemy(type: String, enemy: CharacterBody2D):
    enemy.get_parent().remove_child(enemy)
    _pools.get_or_add(type, []).append(enemy)
```
То же самое для `EggPickup` и `AxeProjectile`.

## 11. Идеи на будущее (Post-MVP)

1. **Кооператив** — 2 игрока на одном экране (локальный) или онлайн через WebSocket.
2. **Ежедневные забеги** — фиксированный сид карты, лидерборд.
3. **Моддинг** — открытые JSON для баланса, поддержка кастомных спрайтов.
4. **Мобильная версия** — виртуальный джойстик, автоматический фаер, короткие забеги (5 мин).
5. **Сезоны** — лето (огород), осень (луг), зима (сугробы, медведи?), весна (паводок).
6. **Торговля** — NPC «Сосед Ваня» продаёт рероллы и бонусы за монеты между забегами.
7. **Секреты на карте** — закопанный сундук (копать 3 сек), случайный бонус.

## 12. Заключение

**«Былины и Мемы»** имеет сильную тематическую основу и харизматичных персонажей. Текущий прототип — хорошая техническая база, но **отсутствие Level-Up экрана, боссов и смерти** превращает игру в бесконечный симулятор, а не в roguelite.

**Приоритет №1:** сделать полный цикл забега (8 минут → босс → супербосс → разблокировка → Game Over → мета-меню).  
**Приоритет №2:** внедрить систему оружия с 6 слотами и эволюциями.  
**Приоритет №3:** подключить оставшихся героев и врагов.

Следуя этой документации, команда из 3 человек (программист + художник + геймдизайнер/звук) может довести игру до Steam-релиза за 2–3 месяца активной работы.
